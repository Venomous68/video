How it works See demo player

Step-by-step

From file to shareable link in 4 steps

01

Create a public repo

Go to GitHub and create a new public repository. Name it something like portfolio-videos or add it to your existing portfolio repo.

02

Push your video file

Commit and push your video (MP4, WebM) to the repo. Keep files under 100MB for direct commits, or use Git LFS for larger files.

git add video.mp4
git commit -m "Add portfolio video"
git push origin main
03

Get the raw URL

On GitHub, click your video file → click the Raw button. Copy that URL — it's a direct link to your video file.

https://raw.githubusercontent.com/
username/repo/main/video.mp4
04

Embed in your portfolio

Use the raw URL as the src in an HTML <video> tag on your portfolio page. Done — instant web player!

<video controls width="100%">
  <source src="YOUR_RAW_URL"
    type="video/mp4">
</video>
01

Create a public project

On GitLab, create a new project with visibility set to Public. GitLab works great for larger files too.

02

Push your video file

Commit and push your MP4/WebM video. GitLab supports files up to 5GB (vs GitHub's 100MB limit), making it ideal for large project videos.

git add video.mp4
git commit -m "Add portfolio video"
git push origin main
03

Get the raw URL

Navigate to your file in GitLab → click Open raw. Copy the URL from the address bar.

https://gitlab.com/username/
repo/-/raw/main/video.mp4
04

Embed in your portfolio

Paste the raw URL into a <video> tag. GitLab serves the file with correct MIME types, so browsers play it natively.

<video controls width="100%">
  <source src="YOUR_RAW_URL"
    type="video/mp4">
</video>

Where to host

Best hosting options for portfolio videos

Recommended

GitHub Raw (+ GitHub Pages)

Push your video to a public repo. GitHub Pages serves your HTML portfolio, and raw.githubusercontent.com serves the video. Both free, same workflow you already use.

  • 100% free, no extra account
  • Version-controlled videos
  • Custom domain support via Pages
  • Up to 100MB per file (Git LFS for more)
Alternative

GitLab CI/CD + Pages

Similar to GitHub but with a 5GB file size limit, making it better for high-res demo videos. GitLab CI can auto-deploy your portfolio on every push.

  • 5GB file size limit (50× GitHub)
  • Free private repos with CI/CD
  • Built-in container registry
  • CORS-friendly raw file serving
External

Cloudflare R2 + Pages

Upload videos to Cloudflare R2 (S3-compatible, free egress), link from your GitHub Pages portfolio. Best for very large or high-traffic video files.

  • Free egress (unlike S3)
  • 10GB free storage
  • Global CDN edge delivery
  • Direct public bucket URLs
Quickest

YouTube / Vimeo (Unlisted)

Upload as unlisted, share only via direct link or embedded iframe. Zero self-hosting setup. Great for very large files or if you want built-in analytics.

  • Zero hosting setup
  • Unlimited file size
  • Built-in player & subtitles
  • Easy iframe embed code
